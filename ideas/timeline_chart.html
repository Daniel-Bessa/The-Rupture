💡 Idea: Per-Fight Timeline Chart

A canvas chart showing DPS Done, Damage Taken, and Healing over the duration of a single boss fight. Placed below each boss's player table on the raid pages.

Status: Parked — The chart renders correctly but the data shape from WCL isn't useful enough at the per-fight level. May revisit as a cross-log comparison chart (see notes below).

The Problem

  • WCL graph API returns cumulative totals per time bucket, not per-second rates.
  • Converting to rates: rate[i] = (cum[i] - cum[i-1]) / interval_s — works but produces very few data points (WCL uses ~20 buckets per fight regardless of duration), making the chart coarse.
  • With only 20 points over a 10-minute fight, each bucket is ~30s wide — not detailed enough to see anything interesting.
  • The chart just shows "DPS went up then stopped" which you already know from the kill.

Better Future Idea: Cross-Log Comparison

  • Compare the same boss across multiple weeks/splits — e.g. average damage taken on Vaelgor week 1 vs week 2 vs week 3.
  • Show improvement (or regression) per player across raids.
  • Bar chart: damage taken per player on a given boss, sorted, with week-over-week delta.
  • This is much more actionable than a single-fight DPS line.

WCL API Notes

# Query used:
query ($code: String!, $fightID: Int!) {
    reportData {
        report(code: $code) {
            dps:   graph(dataType: DamageDone,  fightIDs: [$fightID], hostilityType: Friendlies)
            taken: graph(dataType: DamageTaken, fightIDs: [$fightID], hostilityType: Friendlies)
            heal:  graph(dataType: Healing,     fightIDs: [$fightID], hostilityType: Friendlies)
        }
    }
}

# Response shape:
{
  "data": {
    "series": [
      {
        "name": "PlayerName",
        "type": "Hunter",           # class
        "pointStart": 0,            # ms offset from fight start
        "pointInterval": 42921.75,  # ms per bucket (~20 buckets per fight)
        "total": 14276577,          # total damage over full fight
        "data": [0, 0, ..., 9830, 18343, 33261, 33261]  # CUMULATIVE totals per bucket
      },
      ...
      { "name": "Total", "type": "Total", "data": [...] }  # pre-summed across all players
    ],
    "startTime": 0,
    "endTime": 858000
  }
}

# Conversion to per-second rate (used in wcl_craft_audit.py fetch_fight_graph):
def _sum_series(graph_raw):
    inner   = graph_raw.get("data", graph_raw)
    series  = inner.get("series", [])
    src     = next((e for e in series if e.get("type") == "Total"), None)
    interval_s = src["pointInterval"] / 1000
    prev = 0
    result = []
    for i, cum in enumerate(src["data"]):
        delta = max(cum - prev, 0)
        rate  = round(delta / interval_s)
        t_s   = round((src["pointStart"] + i * src["pointInterval"]) / 1000, 1)
        result.append((t_s, rate))
        if cum > prev: prev = cum
    # trim trailing zeros (fight over)
    while result and result[-1][1] == 0: result.pop()
    return result
